HTML Boilerplate

Individual HTML elements aren't very useful on their own. Next, let's examine how individual elements combine to form an entire HTML page:

Here we have:

  1. Doctype - The doctype. When HTML was young (1991-1992), doctypes were meant to act as links to a set of rules that the HTML page had to follow to be considered good HTML.
  2. html - This element wraps all the content on the page. It is sometimes known as the root element.
  3. head - This element acts as a container for everything you want to include on the HTML page, that isn't the content the page will show to viewers. This includes keywords and a page description that would appear in search results, CSS to style content, character set declarations, and more. You will learn more about this in the next article of the series.
  4. meta - With this setting, the page can now handle any textual content it might contain. There is no reason not to set this, and it can help avoid some problems later.
  5. title - This sets the title of the page, which is the title that appears in the browser tab the page is loaded in. The page title is also used to describe the page when it is bookmarked.
  6. body - This contains all the content that displays on the page, including text, images, videos, games, playable audio tracks, or whatever else.
